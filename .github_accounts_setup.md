# Настройка переключения между GitHub аккаунтами

## Текущая конфигурация

### SSH ключи:
- `qwerpap` (основной) → `~/.ssh/id_ed25519_qwerpap`
- `jjjulietttee-jpg` (новый) → `~/.ssh/id_ed25519_jjjulietttee`
- `gggloriammmartin-jpg` → `~/.ssh/id_ed25519_gggloriammmartin`

### SSH Config (~/.ssh/config):
- `github.com` → использует `id_ed25519_jjjulietttee` (по умолчанию)
- `github.com-qwerpap` → использует `id_ed25519_qwerpap`

## Как переключаться между аккаунтами

### Вариант 1: Для разных проектов (рекомендуется)

Каждый проект использует свой SSH хост, настроенный в remote URL.

#### Для проекта с аккаунтом `qwerpap` (основной):

```bash
# Изменить remote URL на специальный хост
git remote set-url origin git@github.com-qwerpap:qwerpap/repository-name.git

# Настроить Git конфигурацию для этого проекта
git config --local user.name "qwerpap"
git config --local user.email "qwerpap@users.noreply.github.com"
```

#### Для проекта с аккаунтом `jjjulietttee-jpg`:

```bash
# Remote уже настроен на основной хост github.com
git remote set-url origin git@github.com:jjjulietttee-jpg/repository-name.git

# Git конфигурация уже настроена (local)
git config --local user.name "jjjulietttee-jpg"
git config --local user.email "jjjulietttee-jpg@users.noreply.github.com"
```

### Вариант 2: Переключить SSH config по умолчанию

Если хотите, чтобы по умолчанию использовался основной аккаунт `qwerpap`:

1. Измените `~/.ssh/config`:
   ```
   Host github.com
       HostName github.com
       User git
       IdentityFile ~/.ssh/id_ed25519_qwerpap
       IdentitiesOnly yes
   ```

2. Для проектов с новым аккаунтом используйте специальный хост:
   ```bash
   git remote set-url origin git@github.com-jjjulietttee:jjjulietttee-jpg/repository-name.git
   ```

### Вариант 3: Глобальное переключение (менее удобно)

Менять глобальную Git конфигурацию и SSH config каждый раз, когда переключаетесь между проектами.

## Рекомендуемый подход: Настройка для каждого проекта

### Шаг 1: Обновить SSH config для удобства

Добавьте в `~/.ssh/config` отдельные хосты для каждого аккаунта:

```
# Основной аккаунт qwerpap
Host github.com-qwerpap
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_qwerpap
    IdentitiesOnly yes

# Новый аккаунт jjjulietttee-jpg
Host github.com-jjjulietttee
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_jjjulietttee
    IdentitiesOnly yes

# По умолчанию (можно изменить на нужный)
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_qwerpap
    IdentitiesOnly yes
```

### Шаг 2: Для каждого проекта настроить свой remote и Git config

#### Проект для аккаунта qwerpap:
```bash
cd /path/to/project-for-qwerpap
git remote set-url origin git@github.com-qwerpap:qwerpap/repo-name.git
git config --local user.name "qwerpap"
git config --local user.email "qwerpap@users.noreply.github.com"
```

#### Проект для аккаунта jjjulietttee-jpg:
```bash
cd /path/to/project-for-jjjulietttee
git remote set-url origin git@github.com-jjjulietttee:jjjulietttee-jpg/repo-name.git
git config --local user.name "jjjulietttee-jpg"
git config --local user.email "jjjulietttee-jpg@users.noreply.github.com"
```

## Быстрое переключение текущего проекта

### Переключить на основной аккаунт (qwerpap):

```bash
# Изменить remote URL
git remote set-url origin git@github.com-qwerpap:qwerpap/animal_world.git

# Изменить Git конфигурацию
git config --local user.name "qwerpap"
git config --local user.email "qwerpap@users.noreply.github.com"

# Проверить
git remote -v
git config --local --list | grep user
```

### Переключить обратно на jjjulietttee-jpg:

```bash
# Изменить remote URL
git remote set-url origin git@github.com:jjjulietttee-jpg/animal_world.git

# Изменить Git конфигурацию
git config --local user.name "jjjulietttee-jpg"
git config --local user.email "jjjulietttee-jpg@users.noreply.github.com"

# Проверить
git remote -v
git config --local --list | grep user
```

## Проверка подключения

Проверить SSH подключение для каждого аккаунта:

```bash
# Проверка основного аккаунта (qwerpap)
ssh -T git@github.com-qwerpap

# Проверка нового аккаунта (jjjulietttee-jpg)
ssh -T git@github.com-jjjulietttee

# Или через основной хост (зависит от того, какой ключ там настроен)
ssh -T git@github.com
```

---

## Важно

- **Локальная конфигурация** (`--local`) действует только для конкретного репозитория
- **Remote URL** определяет, какой SSH ключ будет использоваться
- Разные проекты могут использовать разные аккаунты одновременно
- Глобальная конфигурация (`--global`) используется как fallback, если локальная не задана

---

## Рекомендация

Лучше всего настроить SSH config с отдельными хостами для каждого аккаунта и использовать локальную Git конфигурацию для каждого проекта. Так вы не будете путаться между аккаунтами.
